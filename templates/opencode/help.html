Описание настроек

Общее

OpenCode AI — это AI-агент для написания, редактирования и понимания кода. Модуль интегрирует opencode в Majordomo, позволяя отправлять запросы AI и получать ответы через чат Majordomo.

Модуль работает через REST API сервера opencode (порт 4096). AI-модели запускаются удалённо на сервере opencode, а модуль отправляет запросы и отображает ответы в интерфейсе Majordomo.

В шапке модуля отображаются цветные индикаторы состояния:

  • зелёный — компонент работает, красный — компонент недоступен.
  • OpenCode — проверяет наличие бинарного файла /usr/local/bin/opencode.
  • API сервер — проверяет доступность сервера opencode через REST API.
  • Sudo — проверяет возможность выполнять команды от root (необходимо для управления systemd-сервисом opencode).
  • Модель — отображает название текущей активной модели. Зелёный — API сервер отвечает, модель подключена. Красный — сервер недоступен.
Модель и агент
Модель
Выбор AI-модели для обработки запросов. Можно выбрать из списка (кнопка "Обновить список моделей" загружает актуальные модели с сервера opencode) или ввести свою модель вручную в формате provider/model_id (например openai/gpt-4o, anthropic/claude-sonnet-4-6).

Если выбрать модель из списка, а затем ввести свою — приоритет у своей модели.
Агент
build (code) — агент для написания и редактирования кода. Умеет создавать, читать и изменять файлы.
general (research) — агент для исследования и поиска информации. Не имеет доступа к файловой системе, предназначен для вопросов и анализа.

Для работы с Majordomo (сценарии, методы) рекомендуется агент build.
Pure Mode
Запуск opencode без внешних плагинов и инструментов. В этом режиме AI отвечает только на основе своей модели, без доступа к файловой системе, интернету или MCP-серверам. Полезен для простых вопросов, не требующих выполнения кода.
Провайдер

Позволяет подключить собственного API-провайдера, если вы не хотите использовать стандартные модели opencode.

API Endpoint
URL API-сервера провайдера. Например: https://api.openai.com, https://api.anthropic.com.
API Key
Ключ API для доступа к провайдеру.
Model Name
Имя модели у провайдера. Например: gpt-4o, claude-sonnet-4-20250514.

Важно: Если поля провайдера пусты, opencode использует модели, настроенные в его конфиге (/var/www/.config/opencode/opencode.jsonc).

MCP серверы

MCP (Model Context Protocol) — протокол, позволяющий AI-модели взаимодействовать с внешними сервисами и инструментами.

MCP Servers (JSON)
JSON-массив для подключения внешних MCP-серверов. Формат: 
[
  {
    "name": "server-name",
    "enabled": true,
    "type": "stdio",
    "command": "python3",
    "args": "-m my_mcp"
  }
]
name — уникальное имя сервера, enabled — включён/выключен, type — тип подключения (stdio/sse), command — команда запуска, args — аргументы через пробел.
Majordomo MCP
Включает встроенный MCP-сервер для управления устройствами Majordomo. AI сможет включать/выключать свет, регулировать температуру и выполнять другие команды через модуль MCP. Отображается только если модуль mcp установлен на сервере.
Системный промпт

Системный промпт — это инструкция, которая задаёт AI контекст и правила поведения. Вы можете указать AI, как он должен отвечать, какие команды выполнять, какой стиль общения использовать.

Пример:

Ты — умный ассистент системы умного дома Majordomo.
Используй [EXEC:команда] для выполнения действий в системе.
Отвечай кратко и по делу. Всегда указывай, какая команда была выполнена.

Промпт применяется ко всем диалогам, пока вы его не измените или не очистите.

Производительность
Max History
Максимальное количество предыдущих сообщений, которые отправляются AI вместе с новым запросом для сохранения контекста беседы. Больше значение = лучше контекст, но выше расход токенов. Рекомендуется 20–50. Старые записи автоматически удаляются в processCycle().
Timeout (секунды)
Максимальное время ожидания ответа от API сервера opencode. Если ответ не получен за это время, curl возвращает ошибку. Для сложных задач (написание кода, анализ проекта) увеличьте таймаут до 120–300 секунд. По умолчанию 120.
Async Timeout (секунды)
Максимальное время ожидания ответа от AI при асинхронной отправке (через чат модуля или COMMAND-событие). Запрос отправляется в фоновый процесс, и модуль ждёт ответ указанное количество секунд. Если AI долго загружается (холодный старт модели), увеличьте это значение. По умолчанию 30. Диапазон 5–120.
Reuse Session
При включении этой опции модуль сохраняет ID сессии в конфиг и использует её для всех последующих запросов. AI "помнит" предыдущие сообщения и контекст беседы сохраняется между разными обращениями. Если опция выключена, каждая отправка сообщения создаёт новую сессию (контекст не сохраняется, но нет накопления токенов).

Если при reuse сессия стала недействительной (404), модуль автоматически создаёт новую. Сессию можно сбросить кнопкой "Очистить" на вкладке "Настройки сервера" → секция "Диагностика".

Автосброс сессии: при смене модели (через список моделей или поле ручного ввода) сессия очищается автоматически — новая модель сразу начнёт отвечать в свежем диалоге.
Отладка (Debug)
При включении в чате модуля отображаются промежуточные статусы обработки запроса: подключение к opencode, ожидание ответа, таймауты. Полезно для диагностики проблем — видно, на каком этапе происходит задержка или ошибка. Статусы отображаются серым текстом внутри сообщения ассистента. Когда ответ готов, статус заменяется ответом AI.
Сеть и авторизация

Настройки сетевого доступа к серверу opencode.

Порт
Порт, на котором работает веб-интерфейс opencode. По умолчанию 4096. Изменяйте, если порт занят другим приложением. После изменения порта все URL (iframe, API endpoint, Open Web) автоматически обновятся.
Авторизация отключена
Opencode web запускается только на 127.0.0.1 (localhost). Доступ к веб-интерфейсу возможен только с самого сервера. Пароль не требуется. Рекомендуется если сервером пользуетесь только вы.
Авторизация включена
Opencode web запускается на 0.0.0.0 (все сетевые интерфейсы). Доступ возможен с любого компьютера в сети по адресу http://IP-СЕРВЕРА:ПОРТ. Для входа потребуется ввести логин и пароль, указанные в настройках.
Логин и пароль
Учётные данные для доступа к веб-интерфейсу opencode. Используются для Basic HTTP-авторизации. С этими же данными модуль обращается к API серверу.
Вкладки настроек

Настройки разделены на две вкладки:

Провайдеры, модели и MCP
Содержит аккордеон из 5 секций: Модель и агент (выбор модели, агента, pure mode), Провайдер (API endpoint, ключ, модель провайдера), MCP серверы (JSON-конфигурация MCP, Majordomo MCP), Системный промпт (инструкция AI) и Производительность (max history, timeout, reuse session, отладка).
Настройки сервера
Содержит аккордеон из 3 секций: Сеть и авторизация (порт, включение авторизации, логин/пароль), Полный доступ (включение/отключение ограничений безопасности AI) и Диагностика системы (путь к бинарнику и конфигу, API endpoint, ID сессии, удаление opencode).

Сохранять настройки можно с любой из этих вкладок — кнопка "Сохранить" находится внизу обеих.

Полный доступ (безопасность)

Управляет уровнем доступа AI-агента к системе:

Полный доступ включён (галочка установлена)
AI работает без ограничений — может читать и редактировать любые файлы, выполнять любые bash-команды, управлять службами. Эквивалентно правам root. Используйте если полностью доверяете AI.
Полный доступ выключен (галочка снята)
AI работает в ограниченном режиме — может читать и редактировать файлы только внутри /var/www/html/cms/files/. Запрещены опасные bash-команды: reboot, shutdown, systemctl, apt, dpkg, pip/npm install, passwd, useradd, mkfs, fdisk, chmod, chown, rm -rf, sudo и другие. Ограничения применяются на уровне конфигурации opencode (permission rules), а не только через системный промпт.
Установка opencode вручную

Если модуль не смог автоматически установить opencode, вы можете установить его вручную:

curl -fsSL https://docs.opencode.ai/install.sh | sh

После установки убедитесь, что бинарный файл находится в /usr/local/bin/opencode (или создайте симлинк):

sudo ln -sf /root/.opencode/bin/opencode /usr/local/bin/opencode

Проверьте установку:

opencode version

Затем настройте провайдера и модель через CLI или直接在 конфиге /var/www/.config/opencode/opencode.jsonc.

Официальная документация: https://opencode.ai/docs

Как пользоваться
Через веб-интерфейс opencode (вкладка "OpenCode Web")

На вкладке "OpenCode Web" отображается iframe веб-интерфейса opencode. Вкладка видна только если включена авторизация (сервер доступен по сети). Кнопка "Открыть Web" открывает интерфейс в новой вкладке.

Через чат модуля в админке

Переключитесь на вкладку "Чат". Введите сообщение в поле ввода и нажмите Enter или кнопку "Отправить". AI ответит в том же окне. Кнопка "Очистить историю" удаляет всю переписку из БД.

Через системный чат (COMMAND event)

Модуль подписывается на событие COMMAND. Когда любой другой модуль или сценарий отправляет команду через callMethod('Commander.command', array('message' => '...')), opencode обрабатывает её и отвечает. Ответ также отправляется через say().

Управление устройствами через MCP

Если включён "Majordomo MCP" (вкладка "Провайдеры, модели и MCP" → секция "MCP серверы"), AI может управлять устройствами умного дома: включать/выключать свет, регулировать температуру, открывать шторы и т.д. Для этого необходим установленный модуль mcp.

Сброс сессии

Если AI начал отвечать неверно или "забыл" контекст — очистите сессию кнопкой "Очистить" на вкладке "Настройки сервера" → секция "Диагностика". После этого начнётся новый диалог. Также можно отключить "Reuse Session" на вкладке "Провайдеры, модели и MCP" → секция "Производительность".

Обновление списка моделей

Кнопка "Обновить список моделей" загружает актуальный список доступных AI-моделей с сервера opencode через GET /provider. Результат кешируется на 1 час. Если API сервер недоступен, используется список из CLI команды opencode models.

Системный промпт по умолчанию

Если системный промпт пуст и включён "Majordomo MCP", модуль автоматически устанавливает промпт для голосового ассистента умного дома на русском языке (на вкладке "Провайдеры, модели и MCP" → секция "Системный промпт").
